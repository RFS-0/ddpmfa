Project

A is a container where you can organize all information of a specific project. In particluar it contains all the models of a project. Before you can create and configure a model you have to create a project. Refer to the quick reference for instructions on how to create a project

Model

The model is the most important entity. Once you have created a project you can create a new model. The configuration of a model is a process that is done in three sequential steps:

  1. Use the Model Designer to construct the material flow you are interested in
  2. Check the parametrization to validate your inputs.
  3. Create an experiment to run the model configuration

You can iterate over this steps as many times as you want.

The model itself is actually a template for your experiments. This allows you to rearrange the entities your model consists of. Each time you create a new experiment the current current state of the model designer and all the parameters of the model will be copied and attached to the experiment. It is important to notice that copy created for the experiment is read only. This ensures that the results of an experiment will always correspond to its design and parameters.

For more details have a look at the description of experiment

Model designer

The model designer is where you can design your model. Here you can add entities and link them with the various kinds of transfers

In particluar you can add the following entites to create you material flow model:

Inflows to the system:

  • External list inflows
  • External function inflows

Compartments of the system:

  • Sink
  • Flow Compartment
  • Stock

You can link those entites with these kinds of transfers:

  • Constant transfer
  • Random choice transfer
  • Stochastic transfer
  • Aggregated transfer

External inflow

External inflows are used to model inflow of material into the system. Ther are two distinct types of external inflows:

  • External list inflows
  • External function inflows

External list inflow

The external list inflow lets you configure a sinlge period inflow for each period of your simulation. Each single period inflow can be one of the following three types:

  • Fixed value inflow
  • Stochastic inflow
  • Random choice inflow

In particular you can edit the following parameters for each external list inflow:

  • Target
  • Name
  • Start delay
  • Derivation distribution
  • Derivation parameters
  • Derivation factor
  • Single periods inflows

Target

The target represents the compartment (i.e. Flow compartment, sink or stock) the material flows to. You can specify only one targe for each external list inflow. Please refer to the quick reference to see how you can specify a target.

Name

The name of the external list inflow.

Start delay

The delay indicates after how many periods of you simulation the external list inflow will start to transfer material. E.g. if your simulation has 10 periods and you configure the start delay to 3, the external list inflow will transfer material from period 3 up to period 10.

Derivation distribution

The derivation distribution is optional.

If a derivation distribution is definted, it is used to sample a derivation factor which in turn is used to determine the derivation factor in each period. The derivation factor is then multiplied by the value the single period inflow of the current period.

Derivation parameters

The derivation parameters are the argument used by the derivation distribution function to determine the derivation factor.

Derivation factor

The derivation factor is used to adjust the material inflow in each period i.e. the inflow value of each period is multiplied by the derivation factor which is being sampled for each period.

Sinlge period inflows

A single period inflow represents the material inflow for a given period. This means that in an external list inflow you have to specify one sinlge period inflow for each period.

Each single period inflow can be of one of three types:

  • Fixed value inflow
  • Stochastic inflow
  • Random choice inflow

Please refer to single period inflow for more details.

External function inflow

The external function inflow lets you define the inflow to the system as a function. For this you have to provide a basic inflow i.e. one of the types of sinlge period inlfows or a number (which then will be converted to a single period inflow).

In particular you can edit the following parameters for each external list inflow:

  • Target
  • Name
  • Basic inflow
  • Inflow function
  • Derivation distribution
  • Derivation parameters
  • Start delay

Target

The target represents the compartment (i.e. Flow compartment, sink or stock) the material flows to. You can specify only one targe for each external list inflow. Please refer to the quick reference to see how you can specify a target.

Name

The name of the external list inflow.

Basic inflow

The basic inflow represents the initial inflow to the system. It is only used in the first period of the simualton. The inflow function uses this inflow to calculate the base value that is then used as an input to the inflow function. It can be either one of the three single period inflow types (i.e. fixed value, stochastic, random choice) or a number that represents the amount material that flows into the system.

Inflow function

The inflow function returns a material amount as inflow for a specific period. The inputs to this function are a basic value and the current period number. If no input function is defined, the basic value is used for all periods.

In particular you can choose between the following inflow function:

  • Linear function

Derivation distribution

The derivation distribution is a probability distribution (from scipy.stats) that is used to represent uncertain knowledge about the true value of the model inflows. The derivation is calculated once per simulation run and applied to the whole inflow list

In particular you can choose between the following probability distributions:

  • Normal distribution
  • Geometric distribution
  • Binomial distribution
  • Exponential distribution
  • Triangular distribution
  • Uniform distributio
  • Gamma distribution
  • Pareto distribution
  • Poisson distribution
  • Chisquare distribution

Derivation parameters

The derivation parameters represent the input values needed to specify the choosen probability distribution (e.g. for a normal distribution: mean, variance).

Start delay

The start delay represents a time lage between the simulation start and the first release from the source

Single period inflow

A single inflow represents the inflow of material to the model in one single period. You can not acctually add a single period inflow to the model since this is an abstract concept that represents the different kinds of inflows you can add to the model. These are in particular:

  • Fixed value inflow
  • Stochastic inflow
  • Random choice inflow

Fixed value inflow

The fixed value inflow represents an inflow amount to a compartment for one specific period.This kind of inflow is used to represent certainty about the true value of this inflow. The fixed value inflow has one parameter:

  • Value

Value

The value represents the inflow value

Stochastic inflow

A stochastic inflow represents an inflow to a compartment of one period. This kind of inflow is used to represent uncertainty about the true value of this inflow. The uncertainty is modeled by a probability distribution function. The stochastic inflow has the following parameters:

  • Probability distribution
  • Probability distribution parameters

Probability distribution

Probability distribution to represent uncertain knowledge about the true value of the inflow

In particular you can choose between the following distributions:

  • Normal distribution
  • Geometric distribution
  • Binomial distribution
  • Exponential distribution
  • Triangular distribution
  • Uniform distributio
  • Gamma distribution
  • Pareto distribution
  • Poisson distribution
  • Chisquare distribution

Probability distribution parameters

Parameter list of the distribution function.

Random choice inflow

A random choice inflow represents an inflow to a compartment of one period. This kind of inflow is used to represent uncertainty about the true value of this inflow. The uncertainty is modeled as a sample representing the assumptions about the true value. For each run, one element of the sample is randomly drawn and used as inflow for the current period.

The random choice inflow has the following parameter:

  • Sample

Sample

Sample to draw a random value from.

Compartment

A compartment is a distinct area of the investigated system. Depending on the scientific question to be aswered with the model, a compartment can be an environmental media, a geographic region or a logical partition in the fate of a material

You can not acctually add a compartment to the model since this is an abstract concept that represents the different kinds of compartments you can add to the model. These are in particular:

  • Flow Compartment
  • Stock
  • Sink

Flow compartment

A flow compartment represents a system compartment without residence time of observed material. The further transfer of material to subsequent compartments is defined using Transfer objects.

In particular you can edit the following parameters for a flow compartment:

  • Name
  • Transfers
  • Log inflows
  • Log outflows
  • Categories

Name

The name of the flow compartment.

Transfers

The transfers represent all outgoing transfers to other compartments

In particular you can choose between the following transfers:

  • Constant transfer
  • Random choice transfer
  • Stochastic transfer
  • Aggregated transfer

Please refer to the transfer section for more details.

Log inflows

Indicates whether the inflows to this flow compartment should be stored or not. If it is not checked, the results will not contain the material inflows of this flow compartment.

Log outflows

Indicates whether the outflows of this flow compartment should be stored or not. If it is not checked, the results will not contain the material outflows of this flow compartment.

Categories

You can define a list of categories the flow compartment is part of (for later evaluation)

Stock

A Stock is a Compartment that accumulates material like sinks do, but it also releases a part of the stored material in later periods. You can specify the material release by configuring a local release.

In particular you can edit the following parameters for a stock:

  • Name
  • Transfers
  • Local release
  • Log inflows
  • Log outflows
  • Categories

Name

The name of the stock.

Transfers

The transfers represent all outgoing transfers to other compartments

In particular you can choose between the following transfers:

  • Constant transfer
  • Random choice transfer
  • Stochastic transfer
  • Aggregated transfer

Please refer to the transfer section for more details.

Local release

A local release defines which proportions of the amount of material stored in the stock are released in which of the subsequent periods. Each stock has exactly one local release.

In particular you can configure the following different types of local releases:

  • Fixed rate release
  • List release
  • Function release

Please refer to the local release section for further details about the individual release types.

Log inflows

Indicates whether the inflows to this stock should be stored or not. If it is not checked, the results will not contain the material inflows to this stock.

Lgo outflows

Indicates whether the outflows of this stock should be stored or not. If it is not checked, the results will not contain the material outflows of this stock.

Categories

You can define a list of categories the flow compartment is part of (for later evaluation)

Sink

A model compartment where inflowing material accumulates i.e. a sink is an endpoint of the material flow.

In particular you can edit the following parameters for a sink:

  • Name
  • Log inflows
  • Categories

Name

The name of the sink.

Log inflows

Indicates whether the inflows to this sink should be stored or not. If it is not checked, the results will not contain the material inflows to this sink.

Categories

You can define a list of categories the sink is part of (for later evaluation)

Local release

You can choose between the following subclasses:

  • List release
