Project

A project is a container where you can organize all information of a specific project. In particluar it contains all the models of a project. Before you can create and configure a model you have to create a project

Model

The model is the most important entity. Once you have created a project you can create a new model. The configuration of a model is process that is done in three sequential steps:

  1. Use the Model Designer to construct the material flow you are interested in
  2. Check the parametrization to validate your inputs.
  3. Create an experiment to run the model configuration

You can of course iterate over this steps as many times as you want.

The model itself is actually a template for your experiments. This allows you to rearrange the entities your model consists of. Each time you create a new experiment the current current state of the model designer and all the parameters of the model will be copied and attached to the experiment. It is important to notice that copy created for the experiment is read only. This ensures that the results of an experiment will always correspond to its design and parameters.

For more details have a look at the description of experiment

Model designer

The model designer is where you can design your model. Here you can add entities and link them with the various kinds of transfers

In particluar you can add the following entites to create you material flow model:

Inflows to the system:

  • External list inflow
  • External function inflow

Compartments of the system:

  • Sink
  • Flow Compartment
  • Stock

You can link those entites with these kinds of transfers:

  • Constant transfer
  • Random choice transfer
  • Stochastic transfer
  • Aggregated transfer

External inflows

External inflows are used to model inflow of material into the system. Ther are two distinct types of external inflows:

  • External list inflows
  • External function inflows

External list inflow

The external list inflow lets you configure one sinlge period inflow for each period of your simulation. Each single period inflow can be one of the following three types:

  • Fixed value inflow
  • Stochastic inflow
  • Random choice inflow

In particular you can edit the following parameters for each external list inflow:

  • Target
  • Name
  • Start delay
  • Derivation distribution
  • Derivation parameters
  • Derivation factor
  • Single periods inflows

Target

The target represents the compartment (i.e. Flow compartment, sink or stock) the material flows to. You can specify only one targe for each external list inflow. Please refer to the quick reference to see how you can specify a target.

Name

The name of the external list inflow.

Start delay

The delay indicates after how many periods of you simulation the external list inflow will start to transfer material. E.g. if your simulation has 10 periods and you configure the start delay to 3, the external list inflow will transfer material from period 3 up to period 10.

Derivation distribution

The derivation distribution is optional.

If a derivation distribution is definted, it is used to sample a derivation factor which in turn is used to determine the derivation factor in each period. The derivation factor is then multiplied by the value the single period inflow of the current period.
